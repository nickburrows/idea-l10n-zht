報告覆寫而不呼叫 super 方法的方法。 這也稱作被拒絕的遺贈。 此類別方法可能表明抽象失敗，導致出現難以追蹤的 bug。

檢查不報告從 java.lang.Object覆寫的方法（clone() 除外）。 按照慣例，clone() 應該呼叫其 super 方法，該方法將返回正確類型的物件。

範例 1：


  class A {
    @Override
    public Object clone() {
      // 不呼叫 'super.clone()'
      return new A();
    }
  }

範例 2：


  interface I {
    default void foo() {}
  }

  class A implements I {
    // 停用“忽略 'default' super 方法”時
    //  對方法發出警告
    @Override
    public void foo(){}
  }

組態檢查：

  • 使用僅在通過以下方式註解 super 方法時報告選項，可以忽略使用所提供列表中的註解標記的 super 方法。 您可以手動將註解新增到此列表中。
  • 使用忽略空 super 方法選項，可以忽略僅拋出異常或空的 super 方法。
  • 使用忽略 'default' super 方法選項，可以忽略接口中的 default super 方法。
