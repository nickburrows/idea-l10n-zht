當在不同 JUnit 版本的測試用例中使用時，報告 JUnit 註解的方法。 為了確定測試用例的框架版本，該檢查會在可用時檢查超類別的框架版本。 當某個超類別不可用時，它將在測試用例中使用最常用的框架。

範例（JUnit 3 測試用例中的 JUnit 4 註解）：


  public class MyTest extends TestCase {
      @Test
      public void foo() { }

      @Test
      @Ignore
      public void testBar() { }
  }

在套用快速修復後：


  public class MyTest extends TestCase {
      public void testFoo() {}

      public void _testBar() {}
  }

範例（JUnit 4 測試用例中的 JUnit 5 註解）：


  public class MyTest {
      @BeforeAll // JUnit 5 生命周期方法
      public void initialize() { }

      @org.junit.Test  // JUnit 4 測試註解
      public void test() {}

      @org.junit.Test  // JUnit 4 測試註解
      public void testWouldBeExecuted() {}
  }

在套用快速修復後：


  public class MyTest {
      @BeforeClass // JUnit 4 生命周期方法
      public void initialize() { }

      @org.junit.Test  // JUnit 4 測試註解
      public void test() {}

      @org.junit.Test  // JUnit 4 測試註解
      public void testWouldBeExecuted() {}
  }
