檢測可能位置錯誤的 Gradle 方法呼叫。

Gradle buildscripts 包含許多嵌套閉包，使得程式碼結構類似於標記語言。 由於 DSL 語言的行為，可以在外部閉包中編寫的方法也可以在內部閉包中使用。 此類別方法在其範圍之外編寫時可能沒有任何意義。
有時可能很難發現這種情況。 此檢查的目的就是發現此類別方法。

範例：


repositories {
  // 'repositories' 的委託具有方法 'exclusiveContent'，可在此處編寫
  maven {
    // 'maven' 的委託具有方法 'content'，可在此處編寫
    // 但也可使用 'exclusiveContent'
    exclusiveContent {} // 報告 'exclusiveContent'
  }
}
