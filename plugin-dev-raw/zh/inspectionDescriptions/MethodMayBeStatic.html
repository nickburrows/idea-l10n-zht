報告可以安全設定為 static 的方法。 盡可能將方法設定為 static 可減少記憶體消耗並提高程式碼品質。

如果滿足以下條件，則方法可以為 static：

  • 不是 synchronized、native 或 abstract，
  • 不參照包含類別中的任何非 static 方法和非 static 欄位，
  • 不是覆寫，也不會在子類別中被覆寫。

使用以下選項組態檢查：

  • 是否只報告 private 和 final 方法，從而提高此檢查的性能。
  • 是否忽略空方法。
  • 使用 Java 8 或更高版本時是否忽略接口中的預設方法。
  • 是否讓該快速修復在呼叫被設為 static 的方法時使用類別參照取代實例限定符，即呼叫 myClass.m() 將被取代為 MyClass.m()。
