當欄位或方法簽名中的類別的可見性不如使用它的成員的可見性時，報告該類的用法。 雖然此類別成員是合法的 Java，但不能在它們參照的類別類型的可見性作用域之外使用。

範例：


  public class Parent {
    public Child getChild() {
      return new Child();
    }

    private class Child {}
  }

此外，在 Java 9 和更高版本中，模組可以通過不匯出它們的軟體套件來向其他模組隱藏它的某些類別。 但是，如果作為匯出的 API 的一部分的成員在其簽名中參照了非匯出類別，則不能在模組外部使用此類別成員。

組態檢查：

  • 使用報告模組 API 中公開的非匯出類別 (Java 9+) 選項可報告公開非匯出類別的模組 API 成員。
    請注意，對於此選項，專案或模組的語言級別必須為 9 或更高。
  • 使用報告在 public API 中公開的不可存取類別選項可報告公開可見性作用域較小的類別的 public 成員。
  • 使用報告在 package-local API 中公開的 private 類別選項可報告公開 private 類別的 package-local 成員。
