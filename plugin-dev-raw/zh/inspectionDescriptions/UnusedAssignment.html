報告賦值後未使用的賦值。 如果不使用，最好移除該賦值，以縮短程式碼並避免冗餘分配。

報告的情況如下：

  • 賦值後從未讀取的變數。
  • 該變數在被讀取前始終被新值覆寫。
  • 變數初始設定式冗餘（出於以上兩種原因之一）。

組態檢查：

使用報告冗餘初始設定式選項可以報告冗餘的初始設定式： 


  int getI() {
    int i = 0; // 冗餘初始化
    i = 2;
    return i;
  }

使用報告可取代為二進制表達式的前綴表達式選項可報告可將 ++i 表達式取代為 i + 1 的情況： 


  int preInc(int value) {
    int res = value;
    return ++res;
  }

使用報告未使用更改值的後綴表達式選項可報告稍後未使用 i 的值的 i++ 情況： 


  int postInc(int value) {
    int res = value;
    return res++;
  }

使用報告其值從未被使用的模式變數選項可報告模式變數的值在讀取之前被覆蓋的情況： 


  if (object instanceof String s) {
      s = "hello";
      System.out.println(s);
  }

使用報告其值從未被使用的迭代參數選項可報告增強的 for 語句的迭代參數的值在讀取前被覆蓋的情況： 


  for (String arg : args) {
      arg = "test";
      System.out.println(arg);
  }
