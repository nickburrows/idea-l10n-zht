報告違反迪米特法則的情況。

迪米特法則並非真正的法則，但它指定了一種樣式準則：絕不呼叫從另一個呼叫中獲取的物件的方法。 遵循此準則的程式碼更容易維護、調整和重構，方法之間的耦合更少，重複更少，資訊隱藏得更好。 另一方面，您可能需要編寫許多包裝器方法才能滿足這一准則。

範例：


  boolean pay(Customer c, Invoice invoice) {
    int dollars = c.getWallet().contents; // 違規
    if (dollars >= invoice.getAmount()) {
      Wallet w = c.getWallet();
      w.subtract(invoice.getAmount()); // 違規
      return true;
    }
    return false;
  }
上面的範例可以更好地實作為 Customer 中的 payInvoice(Invoice invoice) 方法。

使用忽略對庫方法的呼叫和對庫欄位的存取選項可以忽略不更改庫就無法修正的違反迪米特法則的情況。
