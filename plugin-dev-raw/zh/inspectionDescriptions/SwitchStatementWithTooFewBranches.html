報告帶有太少 case 標籤的 switch 語句和表達式，並建議將它們覆寫為 if 和 else if 語句。

範例 (minimum branches == 3)：


  switch (expression) {
    case "foo" -> foo();
    case "bar" -> bar();
  }

在套用快速修復後：


  if ("foo".equals(expression)) {
    foo();
  } else if ("bar".equals(expression)) {
    bar();
  }

不報告詳盡的 switch 表達式 (Java 14+) 或不帶 'default' 分支的模式 switch 語句（Java 17 預覽版）。 這是因為當 switch 被轉換為 if 後，編譯時詳盡性檢查將丟失（可能是不希望發生的結果）。

組態檢查：

使用最小分支數 欄位指定 case 標籤的最小預期數量。

使用不報告模式 switch 語句選項可避免報告具有模式分支的 switch 語句和表達式。 例如：


  String result = switch(obj) {
    case String str -> str.trim();
    default -> "none";
  };

即使只有一個模式分支，也應該保留 switch，而不是使用 instanceof 語句。
