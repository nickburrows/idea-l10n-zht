報告會在 Kotlin 1.8 及更高版本中導致編譯錯誤的 when 分支中的模糊邏輯表達式。


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      this in (4..7) -> true // 不明確
      else -> false
  }

在套用快速修復後：


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      (this in (4..7)) -> true // 包裝在圓括號中
      else -> false
  }

從 1.7 版開始，可對 Kotlin 語言級別進行檢查。
