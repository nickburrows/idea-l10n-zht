報告在物件建構期間對當前類別的覆寫方法的任何呼叫。 如果物件建構在內部，則會發生這種情況：
  • 建構函式
  • 非 static 實例初始設定式
  • 非 static 欄位初始設定式
  • clone()
  • readObject()
  • readObjectNoData()

此類別呼叫可能會導致難以捉摸的錯誤，因為不能保證物件在方法呼叫發生之前被初始化。

範例：


  abstract class Parent {
    void someMethod() { }
  }

  class Child extends Parent {
    Child() {
      someMethod();
    }

    @Override
    void someMethod() { }
  }

此檢查與以下各項共享其功能：

  • 物件建構期間呼叫的 abstract 方法檢查
  • 物件建構期間呼叫的可覆寫方法檢查

一次只應啟用一個檢查，以防止重複警告。
