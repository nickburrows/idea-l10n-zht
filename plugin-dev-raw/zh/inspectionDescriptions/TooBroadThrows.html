報告異常比方法實際拋出的異常更通用的 throws 子句。

範例：


  public void createFile() throws Exception { // 警告：'throws Exception' 範圍過廣，屏蔽異常 'IOException'
    File file = new File("pathToFile");
    file.createNewFile();
  }

在套用快速修復後：


  public void createFile() throws IOException {
    File file = new File("pathToFile");
    file.createNewFile();
  }

組態檢查：

  • 使用要警告的隱藏異常的最大數量欄位可以忽略隱藏了比指定數量更多的其他異常的異常。
  • 使用僅對 RuntimeException、異常、錯誤或 Throwable 發出警告選項可以使此檢查僅對最常見的異常發出警告。
  • 使用忽略在覆寫庫方法的方法上宣告的異常選項可以忽略在覆寫庫方法的方法中過於寬泛的 throws 子句。
  • 使用忽略隱藏其他異常但自身被拋出的異常選項可以忽略任何隱藏其他異常，但仍可能從方法體拋出並因此在技術上不會過於寬泛的異常。
