報告從 1.9 版開始會導致編譯錯誤的由 @Deprecated 註解傳播的宣告。

動機類型：

  • 出於實作設計/架構原因，需要對實作進行更改
  • 設計中的不一致（在不同的上下文中存在不同的處理方式）

更多詳細資訊：KT-47902：不要通過覆寫傳播方法棄用

該快速修復會複製父宣告中的 @Deprecated 註解。

範例：


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      override fun foo() {}
  }

在套用快速修復後：


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      @Deprecated("Don't use")
      override fun foo() {}
  }

僅當專案或模組的 Kotlin 語言級別為 1.6 或更高時，此檢查才會報告。
