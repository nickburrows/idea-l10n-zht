報告對 StringBuilder、StringBuffer 和 java.util.concurrent.atomic 軟體套件的實例的 equals() 呼叫。

equals() 方法在這些類別中沒有被覆寫，因此即使兩個物件的內容相同，它也可能返回 false。 如果您打算使用參照相等，我們建議您使用 == 以避免混淆。

StringBuilder、StringBuffer、AtomicBoolean、AtomicInteger、AtomicBoolean 和 AtomicLong 的快速修復可用來轉換為內容的比較。 當其中一個實例為 null 時，快速修復可能會改變語意。

範例：


  public void test(StringBuilder sb1, StringBuilder sb2) {
    boolean result = sb1.equals(sb2); // 可疑
  }

在套用快速修復後：


  public void test(StringBuilder sb1, StringBuilder sb2) {
      boolean result = sb1.toString().equals(sb2.toString());
  }
