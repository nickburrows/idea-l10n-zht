報告可以安全地取代為 static 內部類別的匿名類別。 如果某個匿名類別沒有從周圍方法顯式參照其封閉實例或區域類別，則它可能是 static 內部類別。

static 內部類別不保留對其封閉實例的隱式參照。 這可以防止常見的記憶體泄漏，並減少每個類別實例使用的記憶體。

從 Java 18 開始，只有可序列化的匿名類別在不使用該參照的情況下，才保留對其封閉實例的隱式參照。 因此，如果模組語言級別為 Java 18 或更高，則此檢查只報告可序列化的類別。

該快速修復會將匿名類別提取到命名的 static 內部類別中。

範例：


  void sample(){
    Thread thread = new Thread(new Runnable() {
      @Override
      public void run() {
      }
    });
  }

在套用快速修復後：


  void sample() {
    Thread thread = new Thread(new Task());
  }

  private static class Task implements Runnable {
    @Override
    public void run() {
    }
  }
